Expression Editor Predefined Functions

The Expression Editor Window provides predefined expressions for the user. Below are the predefined functions provided in the Expression Editor.

  • Common Functions
  • Date Functions
  • Force Error Functions
  • Numeric Functions
  • String Functions
  • Validations Functions

Common Functions

1. boolean vectorCompare(ReusableRow arg0, String arg1, String arg2)

This method can be used to simulate the vector comparison like old == new, where 'old' and 'new' are vectors with equal number of sub-fields in them. The field names and data types should match for a correct comparison.

  • Parameters:
    • inputRow the ReusableRow object whose fields are to be compared
    • prefix1 the prefix of the vector 1 to be compared
    • prefix2 the prefix of the vector 2 to be compared
  • Returns:
    • true if all the field values match
    • false is any field value does not match

Date Functions

1. Integer today()

Returns number of days since 1st January 1900.

  • Returns:
    • number of days since 1st January 1900

2. TT now()

Returns current date and time in yyyyMMddHmmssS format

  • Returns:
  • current date and time in yyyyMMddHmmssS format

3. String dateFormatter(String inputValue, String oldFormat, String newFormat )

Formats and converts a date value into string representation in the desired new date format.

  • Parameters:
    • inputValue the date value in old format
    • oldFormat the date format of the date passed in inputValue
    • newFormat the desired date format
  • Returns:
    • string representation of date in new date format
    • the method returns null if any parameter is null
  • Throws:
    • ParseException if the date value passed in inputDateInStringFormat does not match the dateFormat

Throws a runtime exception with the error message.

Numeric Functions

1. String decimalStrip(String inputValue)

Retains just the decimal numbers 0-9 excluding the decimal point from the inputValue.

  • Parameters:
    • inputValue the value from which the decimals are to be retained
  • Returns:
    • the decimals from the inputValue

2. String decimalStrip(String inputValue, String decimal_point)

Retains just the decimal numbers 0-9 including decimal point as specified in decimalPoint from the inputValue.

  • Parameters:
    • inputValue the value from which the decimals are to be retained
    • includeDecimalPoint
  • Returns:
    • the decimals from the inputValue

3. T mathAbs(T inputValue)

Returns the absolute value of the argument.

  • Parameters:
    • inputValue whose absolute value is to be determined
  • Returns:
    • the absolute value of the argument

4. Integer random(T n)

Returns a pseudorandom, uniformly distributed int value between 0 (inclusive) and the specified value (exclusive), drawn from this random number generator's sequence.

  • Parameters:
    • n the bound on the random number to be returned. Must be positive.
  • Returns:
    • the next pseudorandom, uniformly distributed int value between 0 (inclusive) and n (exclusive) from this random number generator's sequence

5. Double getDoubleFromComparable(T inputValue)

Returns a double value.

    • tgtString the target string to replace in inputValue
  • Returns:

    • inputValue the value in which the source string is to be replaced with target string
