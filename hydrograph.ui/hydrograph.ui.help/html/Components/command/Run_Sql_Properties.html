Run Sql Properties

Properties for the Run Sql component can be viewed by Double click component on canvas. The properties contain a 'General' tab. Common and mandatory properties are present in the General tab.

General Properties

Details

  • Name - The Name field will specify the name of the component as desired by the user.
  • ID - ID field will specify unique id for every component.
  • Type - Type further categorizes the component to detailed operation that it intends to perform. Here it is of type command Run Sql.

Configuration

  • Database - Database Name used to specify to select database from combo box. By default this is Oracle.

    • if user selects "Parameter", provide a textbox where user can input the Database Name.

  • Database Name - Database Name used to specify the database. This field should be mandatory to user.
  • Host Name - Host Name used to specify the Host server IP address or localhost name. This field should be mandatory to user.
  • Port - Port simply accepts a numeric number starting from 0 to maximum 65535. By default this is 1521.
  • User - User used to specify the User name Logon to specify Database.
  • Password - Password used to specify the password.
  • Sql Query - Sql Query used to specify a multiline text box where user can type the query to be executed
  • Batch - Batch simply accepts a numeric number starting from 0 to maximum 99 and signifies the batch this component will execute in. By default this is 0.

Validations

The Run Sql components applies validations to the mandatory fields. Upon placing the Run Sql component on job canvas for the first time (from component palette), the component shows up a warning icon as mandatory properties are not provided.

The properties window also displays error icon on mandatory fields if it has an incorrect value. The error icon is displayed on the tab as well, if any of the field within the tab has some error.

If the properties window has some error even after user visit's it once, then the warning icon on the Run Sql component on the job canvas changes to error icon. This error icon is removed only when all the mandatory fields are supplied with correct values.
