View Execution History

View Execution History  is a replay feature for displaying execution status of specified job execution, without re-running the job itself.

To View/Watch Execution History:

  • To view Execution History the job should be run at least once.

  • If the job is never executed then you get a message saying No execution history available for this job in current session.

  • View Execution History feature is added on drop down of RUN button as shown below.

    • Fig: Click the View Execution History option to see run history.

  • Once you select View Execution History option from run button drop down a window would open ,showing history of previous job executions.

Fig: History for previous job executions.

  • The View Execution History window shows different parameters for previous runs

    • Job Id: Unique Job Id is common for all runs for that particular job.Currently, when you create and save the job, unique job Id is getting generated and is saved into its xml.Unique Job ID is a combination of Job_RandomUniqueNumber_LongTimestamp. Ex. JobName="job_1.xml" UniqueJobId="Job_1924263499_1487226195589" Throughout, this Job id will remain same, even if you make modifications to same job and take multiple runs. So this same job id will be used in tracking logs.

      To identify different run for same job we append time stamp to UniqueJobId

    • Time stamp: This is time of the job run.

    • Execution mode: Execution mode can be either Local or Remote

    • Job status: This is the status of job,it can either be KILLED, FAILED or SUCCESS

  • If user wants to view the run history for any past run, user just needs to browse the required tracking log file using browse button.

Fig: Browse older execution files to see execution history form past.

  • The execution files have .track.log file extension, so user is able to browse files only with this extension.

  • If user selects the job tracking file with incorrect file , error message is displayed saying Incorrect format

Fig: Message when incorrect file is selected.

  • If some components and ports of existing components were added/removed the record count or status for those component and ports will not be displayed and such components and port details will be listed on a window.

Fig: Message when some components were added/removed.
